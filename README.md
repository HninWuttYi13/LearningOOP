##  Key Concepts Learned

- **Encapsulation** → control access to data  
- **Inheritance** → reuse and extend behavior  
- **Polymorphism** → flexible behavior  
- **Abstraction** → simplify complexity  
- **Prototypes (JS)** → shared behavior mechanism  

---

##  Common Mistakes I Faced

- Misusing `this` (especially with arrow functions)  
- Forgetting `new` in constructor functions  
- Putting methods inside constructors (memory inefficiency)  
- Confusing real-world objects with software design  
- Overusing inheritance instead of composition  

---

##  Key Takeaways

- “Constructor is for data, prototype is for behavior”  
- “`this` depends on how a function is called”  
- “Class is syntactic sugar over prototypes in JavaScript”  
- “Encapsulation is about control, not just hiding”  
- “Composition is often better than inheritance”
